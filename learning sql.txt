	У ситуаціях, коли у таблиці є кілька повторюваних записів, може бути доцільніше повертати лише унікальні записи, а не отримувати повторювані записи. Ключове слово SQL DISTINCT використовується в поєднанні з SELECT, щоб видалити всі повторювані записи та повернути лише унікальні.

SELECT DISTINCT column_name1, column_name2
FROM table_name;
-------------------------------------------------
	За замовчуванням повертаються всі результати, які задовольняють умовам, зазначеним у операторі SQL. Однак іноді нам потрібно отримати лише підмножину записів. У MySQL це досягається за допомогою ключового слова LIMIT.

SELECT column list
FROM table_name
LIMIT [number of records];
-------------------------------------------------
	Ви також можете підібрати набір записів з певного зміщення.У наступному прикладі ми вибираємо чотири записи, починаючи з третьої позиції:

SELECT ID, FirstName, Last name, City
FROM customers OFFSET 3 LIMIT 4;
-------------------------------------------------
	У SQL ви можете вказати ім’я таблиці перед назвою стовпця, розділивши їх крапкою.

SELECT City FROM customers;
*
SELECT customers.City FROM customers;

За замовчуванням ключове слово ORDER BY сортує результати в порядку зростання.
-------------------------------------------------
	ORDER BY використовується разом із SELECT для сортування повернених даних.У наступному прикладі таблиця наших клієнтів сортується за стовпцем FirstName.

SELECT * FROM customers
ORDER BY FirstName;
-------------------------------------------------
	Сьогодні клієнт хоче мати торт з мінімальною кількістю калорій.Напишіть запит, щоб відсортувати торти за кількістю калорій, і виберіть перші 3 торти зі списку, щоб запропонувати клієнту.

SELECT * FROM cakes
ORDER BY calories LIMIT 3;
-------------------------------------------------
Речення WHERE використовується для вилучення лише тих записів, які відповідають визначеному критерію.

SELECT column_list 
FROM table_name
WHERE condition;

Оператори порівняння та логічні оператори використовуються в пропозиції WHERE для фільтрації даних, які потрібно вибрати

SELECT * FROM customers
WHERE ID != 5;
-------------------------------------------------
Оператор BETWEEN вибирає значення в межах діапазону. Перше значення має бути нижньою, а друге значення — верхньою.Синтаксис пропозиції BETWEEN такий:

SELECT column_name(s)
FROM table_name
WHERE column_name BETWEEN value1 AND value2;
-------------------------------------------------
Працюючи з текстовими стовпцями, візьміть будь-який текст, що з’являється в інструкції, одинарними лапками (').Наступний оператор SQL вибирає всі записи, у яких місто дорівнює "Нью-Йорк".

SELECT ID, Firstname, LastName, City
FROM customers
WHERE City = 'New York';
